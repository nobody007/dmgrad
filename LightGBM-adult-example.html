LightGBM Practice: Adult Income Data

Course: Datamining — Graduate course  |  Instructor: Jongwoo Song

1. Introduction

LightGBM (Light Gradient Boosting Machine) is a fast and efficient gradient boosting framework developed by Microsoft. Compared to XGBoost, LightGBM uses leaf-wise (best-first) tree growth instead of level-wise growth, which leads to faster training and often better accuracy on large datasets.

Key features of LightGBM:

  • Leaf-wise tree growth (lower loss than level-wise)
  • Histogram-based algorithm (fast, memory-efficient)
  • Native support for categorical features
  • Built-in cross-validation and early stopping

In this example we use the Adult Income dataset (UCI Machine Learning Repository) to predict whether a person earns more than $50K/year. This is a binary classification problem with both numeric and categorical features.

4. Data Preprocessing

# Remove rows with missing values
train_raw <- na.omit(train_raw)
test_raw  <- na.omit(test_raw)

# Fix test set income labels (has trailing period: "<=50K.")
test_raw$income <- gsub("\\.", "", test_raw$income)

# Encode target: >50K = 1, <=50K = 0
train_raw$y <- as.integer(train_raw$income == ">50K")
test_raw$y  <- as.integer(test_raw$income  == ">50K")

# Drop original income column and fnlwgt (sampling weight, not a predictor)
drop_cols <- c("income", "fnlwgt")
train_raw <- train_raw[, !names(train_raw) %in% drop_cols]
test_raw  <- test_raw[,  !names(test_raw)  %in% drop_cols]

# Identify categorical columns and convert to integer codes
# LightGBM requires categorical features to be integer-encoded
cat_cols <- c("workclass", "education", "marital_status",
              "occupation", "relationship", "race",
              "sex", "native_country")

for (col in cat_cols) {
  lvls <- unique(c(train_raw[[col]], test_raw[[col]]))
  train_raw[[col]] <- as.integer(factor(train_raw[[col]], levels = lvls)) - 1L
  test_raw[[col]]  <- as.integer(factor(test_raw[[col]],  levels = lvls)) - 1L
}

5. Create LightGBM Datasets

# Separate features and labels
feature_cols <- setdiff(names(train_raw), "y")

X_train <- as.matrix(train_raw[, feature_cols])
y_train <- train_raw$y

X_test <- as.matrix(test_raw[, feature_cols])
y_test <- test_raw$y

# Build LightGBM dataset objects
dtrain <- lgb.Dataset(X_train, label = y_train,
                       categorical_feature = cat_cols)
dtest  <- lgb.Dataset(X_test,  label = y_test,  reference = dtrain)

6. Train LightGBM Model

params <- list(
  objective        = "binary",
  metric           = c("binary_logloss", "auc"),
  learning_rate    = 0.05,
  num_leaves       = 31,       # max leaves per tree (controls complexity)
  max_depth        = -1,       # -1 means no limit (num_leaves controls this)
  min_data_in_leaf = 20,
  feature_fraction = 0.8,      # subsample features per tree (like colsample_bytree)
  bagging_fraction = 0.8,      # subsample rows per iteration
  bagging_freq     = 5,
  verbose          = -1
)

set.seed(42)
model <- lgb.train(
  params    = params,
  data      = dtrain,
  nrounds   = 500,
  valids    = list(train = dtrain, test = dtest),
  early_stopping_rounds = 30,   # stop if no improvement for 30 rounds
  verbose   = 50
)

9. Cross-Validation with lgb.cv

set.seed(42)
cv_result <- lgb.cv(
  params   = params,
  data     = dtrain,
  nrounds  = 300,
  nfold    = 5,
  early_stopping_rounds = 20,
  verbose  = -1
)

10. LightGBM vs XGBoost: Key Differences

Aspect LightGBM XGBoost
Tree growth strategy Leaf-wise (best-first) Level-wise (depth-first)
Speed Faster (histogram-based) Slower on large data
Memory usage Lower Higher
Categorical features Native support Requires manual encoding
Overfitting risk Higher (leaf-wise) Lower (level-wise)
Key hyperparameter num_leaves max_depth
Adult data AUC (approx.) 0.927 0.924
Note: On this dataset the difference is small, but LightGBM trains noticeably faster. The advantage becomes larger with millions of rows.
